## Task 1 — 前端 API URL 改用環境變數

**問題：** `app-front/src/App.tsx` 的 `sendMessage()` 中，後端 URL `http://127.0.0.1:8000/api/v1/chat/stream` 為硬編碼。

**解決方案：** 使用 Vite 環境變數 (`VITE_` 前綴) 取代硬編碼。

**變更清單：**

| 檔案 | 動作 |
|------|------|
| `app-front/.env` | 新增，設定 `VITE_API_BASE_URL=http://127.0.0.1:8000` |
| `app-front/.env.example` | 新增，提交至版本控管供團隊參考 |
| `app-front/.gitignore` | 加入 `.env` / `.env.local` 排除規則 |
| `app-front/src/App.tsx` | 硬編碼 URL → `` `${import.meta.env.VITE_API_BASE_URL}/api/v1/chat/stream` `` |

## Task 2 — 將前端 Menu 設定移至後端 API

**問題：** `app-front/src/components/layout/nav-config.ts` 的 `navGroups` 靜態資料寫死在前端，無法由後端動態管理。

**核心挑戰：** `NavItem.icon` 為 React 元件 (`LucideIcon`)，無法序列化，後端改以 icon string name 回傳，前端維護 string → LucideIcon 對照表。

**變更清單：**

| 檔案 | 動作 |
|------|------|
| `app-backend/app/schemas/menu.py` | 新增 `NavItemSchema` / `NavGroupSchema` Pydantic models |
| `app-backend/app/api/v1/routes/menu.py` | 新增 `GET /api/v1/menu` endpoint，選單資料由此管理 |
| `app-backend/app/main.py` | 註冊 `menu.router` |
| `app-front/src/components/layout/nav-config.ts` | 保留執行期介面；新增 `NavItemAPI` / `NavGroupAPI` wire types 與 `ICON_MAP`；移除靜態 `navGroups` |
| `app-front/src/hooks/useNavGroups.ts` | 新增 hook，從後端 fetch 並將 icon string 轉換為 `LucideIcon` |
| `app-front/src/components/layout/Sidebar.tsx` | 改用 `useNavGroups()` hook；支援 loading / error 狀態 |

**API 端點：**
```
GET /api/v1/menu
Response: NavGroupSchema[]
```
