## 任務二：實作 tool_commands dispatch 機制

**來源：** `tasks/task-04-17/解決方案.md`

### 問題

`_tool_commands` 載入為 `set[str]`，丟失 `action` 欄位，且 `_detect_clear` 為寫死邏輯，完全未使用 `_tool_registry`。

### 解決方案

將 `_tool_commands: set[str]` 改為 `_tool_registry: dict[str, str]`（`command → action`），以通用 dispatch 函式取代寫死的 `_detect_clear`。

### 實作內容

**`chat.py` 架構：**
```
_tool_registry: dict[str, str]       # command → action（來自 tools.json）
_ACTION_HANDLERS: dict[str, ...]     # action → handler
_dispatch_command(messages)          # 查表並執行 handler
```

**呼叫鏈（以 /clear 為例）：**
```
"/clear" → _tool_registry → "clear_messages" → _ACTION_HANDLERS → "對話已清除。"
```

## 任務三：新增 /webfetch [url] 動態指令

### 設計變更

| 項目 | Before | After |
|------|--------|-------|
| `_ACTION_HANDLERS` 型別 | `dict[str, str]` | `dict[str, Callable[[str], Awaitable[str]]]` |
| `_dispatch_command` | 同步、exact match | async、前綴比對 + args 解析 |

### 實作內容

**`tools.json`** 新增：
```json
{"command": "/webfetch", "description": "抓取指定 URL 的網頁內容", "action": "web_fetch"}
```

**`chat.py`** 新增 `_handle_web_fetch`：
- 使用 `httpx.AsyncClient` 抓取 URL
- 回傳前 3000 字元（避免超出 context 限制）
- 錯誤處理：HTTP 狀態錯誤、連線失敗

**使用方式：**
```
/webfetch https://example.com
```

### 最終檔案狀態

- `app-backend/system-env/tools.json`：含 `/clear`、`/webfetch` 兩個指令
- `app-backend/app/api/v1/routes/chat.py`：完整 async dispatch 機制
- `app-front/src/App.tsx`：`/clear` 後正確清除 messages state
