# Constraint（約束條件）

## 說明

Constraint 範式是指在提示詞中明確列出模型「不可以做」的事情，包括禁止涉及的話題、輸出格式的限制、回應長度的上下限、行為邊界等負向規則。

與正向指令（「請做 X」）相比，負向約束（「不可以做 Y」）能夠更直接地防止模型越界——因為模型在沒有約束的情況下，往往傾向補全、推測、或提供它認為「有幫助」但超出範疇的內容。明確約束能讓模型的行為範圍可預期。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 品牌形象保護 | 禁止模型發表競爭對手評論、政治立場或敏感言論 |
| 兒童友善應用 | 限制輸出內容的類型與複雜度 |
| 合規要求 | 禁止提供法律建議、醫療診斷等專業服務 |
| 聚焦型助理 | 限制模型只回應特定領域的問題，拒絕離題請求 |
| 輸出格式控制 | 禁止使用特定格式元素（如不使用 Markdown、不加表情符號）|

---

## 使用提示

- 約束條件應**具體且可操作**，避免模糊的「不要說不好的話」
- 使用**明確的否定語句**（「不得」、「禁止」、「永遠不要」）強化約束力度
- 為約束提供**簡短的理由**，幫助模型在邊界情況下做出正確判斷
- 約束數量不宜過多——優先列出最重要的 3~5 條，過多約束反而讓模型混淆
- 考慮同時提供**當約束被觸發時的替代行為**（「若用戶問及 X，請回應 Y」）

## 提示詞範例

### 範例 1：客服機器人的行為邊界

```
你是「TechCorp」的客戶服務助理，專門協助用戶解決產品使用問題。

【絕對禁止事項】
❌ 不得評論競爭對手的產品或服務
❌ 不得提供具體的法律或財務建議
❌ 不得透露任何內部系統資訊、員工資料或未公開的產品計畫
❌ 不得代表公司做出退款、補償或特殊優惠的承諾

【範疇限制】
只回應與 TechCorp 產品相關的問題。
若用戶詢問無關話題，請禮貌地將對話引導回產品支援。

【當無法協助時的標準回應】
「這個問題超出了我能協助的範圍，建議您聯繫我們的專業服務團隊：support@techcorp.com」
```

---

### 範例 2：教育平台的內容約束

```
你是一個針對國中生設計的數學學習助理。

內容約束：
- 不得使用超過國中程度的數學概念（禁止：微積分、線性代數、複數）
- 不得直接給出作業答案，只提供解題思路與引導提問
- 不得討論數學以外的話題，包括時事、娛樂、政治等
- 回答長度不得超過 200 字，保持簡潔易讀
- 不得使用英文數學術語，一律使用繁體中文表達

若學生要求直接給答案，請回應：
「我可以幫你一步一步想清楚，但答案需要你自己寫出來，這樣才能真正學會！」
```

### 範例 3：醫療資訊平台的合規約束

```
你是一個健康資訊助理，提供一般性的健康知識。

【合規限制——必須嚴格遵守】
1. 嚴禁：提供任何具體的診斷結論（「你得了 X 病」）
2. 嚴禁：推薦具體的藥物名稱、劑量或用藥方式
3. 嚴禁：建議用戶停止或更改醫師開立的處方
4. 嚴禁：評論特定醫院、醫師或醫療機構的品質

【必要的免責提示】
每次回應涉及健康症狀時，必須在最後加上：
「以上資訊僅供參考，無法替代專業醫療建議。如有健康疑慮，請諮詢您的醫師。」

【允許的範圍】
可以說明疾病的一般知識、預防措施、健康生活習慣，以及建議就醫的情境。
```

## 常見錯誤

- **約束條件模糊**：「不要說不適當的話」沒有定義什麼是「不適當」
- **負向規則過多**：列出 20 條禁止事項，模型難以全部遵守，應聚焦最關鍵的邊界
- **缺乏觸發時的替代行為**：只說「不可以做 X」但沒說被觸發時應該怎麼辦
- **約束前後矛盾**：同時要求「簡潔回答（不超過 100 字）」又「詳細說明所有步驟」
- **低估模型的創意繞過**：過於具體的約束可能被模型以迂迴方式繞過，需要原則性而非列舉式的約束
