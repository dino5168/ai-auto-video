# Few-Shot Calibration（少樣本校準）

## 說明

Few-Shot Calibration 範式是指在提示詞中精心挑選並提供少量示例（通常 2~5 個），以校準模型的輸出分佈，使其符合特定的格式、風格、推理模式或品質標準。

與一般示例不同，Few-Shot Calibration 的重點在於**策略性選例**——示例的選擇本身就是在傳達「好的輸出應該長什麼樣」，而不只是單純示範任務如何進行。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 難以用文字精確描述的輸出風格 | 「示範」比「描述」更有效傳達期望 |
| 需要高度一致的輸出格式 | 確保每次輸出都符合特定模板 |
| 邊界案例處理 | 用示例展示特殊情況的正確處理方式 |
| 分類或標注任務 | 用示例定義各類別的判斷標準 |
| 推理模式引導 | 展示期望的思考鏈結構（Chain of Thought） |

---

## 使用提示

- **示例品質優先於數量**：2 個高品質示例優於 5 個普通示例
- **涵蓋邊界案例**：至少包含一個需要判斷的非典型案例
- **示例應覆蓋不同變體**：避免所有示例都是同一類型
- **保持示例與任務的一致性**：示例的輸入格式應與實際輸入相同
- **避免示例偏差**：若所有示例都偏向某一結果，模型會學到偏見

## 提示詞範例

### 範例 1：情感分析標注校準

```
請分析以下評論的情感傾向。

示例：
輸入：「產品品質很好，但客服態度讓我很失望。」
輸出：混合（正面：品質；負面：客服）

輸入：「比預期便宜，功能也夠用，不會後悔買。」
輸出：正面（低期望被超越）

輸入：「就那樣，沒什麼特別的。」
輸出：中性（無明顯情感傾向）

---

請分析：
「包裝很精美，可惜東西跟圖片差很多，有點失落。」
```

### 範例 2：命名規範校準

```
請為以下函式產生符合規範的命名建議。

示例：
功能：取得使用者的所有訂單
建議命名：getUserOrders、fetchUserOrderList

功能：將金額從台幣轉換為美金
建議命名：convertTWDToUSD、exchangeCurrencyToUSD

功能：驗證電子郵件格式是否正確
建議命名：validateEmailFormat、isValidEmail

---

請為以下功能命名：
功能：檢查使用者是否擁有特定權限
```

### 範例 3：Chain of Thought 推理校準

```
請解答以下數學應用題，並展示推理過程。

示例：
問題：小明有 3 包糖果，每包 12 顆，送給同學 15 顆後剩幾顆？
推理：
1. 計算總糖果數：3 × 12 = 36 顆
2. 計算剩餘：36 - 15 = 21 顆
答案：21 顆

問題：一輛車以時速 60 公里行駛，2.5 小時後距離起點多遠？
推理：
1. 套用公式：距離 = 速度 × 時間
2. 計算：60 × 2.5 = 150 公里
答案：150 公里

---

請解答：
問題：一家店原售價 800 元的商品打八折後，再折抵優惠券 50 元，最終售價為多少？
```

## 常見錯誤

- **示例品質低落**：示例本身有錯誤或不具代表性，反而誤導模型
- **示例格式不一致**：示例的輸入/輸出格式與實際任務格式不同
- **示例數量過多**：超過 5~6 個示例效益遞減，且佔用大量 token
- **示例覆蓋不足**：所有示例都是簡單案例，無法校準邊界情況的處理
