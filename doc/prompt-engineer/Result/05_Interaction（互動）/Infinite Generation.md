# Infinite Generation（無限生成）

## 說明

Infinite Generation 範式是指設計一種可以無限延伸的生成結構，讓模型能夠在每次呼叫時產生新的、連貫的內容片段，並在需要時無縫銜接前一段輸出，適用於持續性創作、串流輸出、或需要動態擴展的內容場景。

這個範式的核心機制是：每次生成時傳入前一段輸出作為「接續點」（continuation seed），讓模型知道從哪裡繼續，同時確保整體的一致性與連貫性。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 長篇小說或故事創作 | 分批生成章節，保持情節與角色的一致性 |
| 新聞或報告的持續更新 | 每次有新資訊時，在現有報告末尾追加更新 |
| 程式碼生成 | 超出 context 限制的大型程式碼，分批生成 |
| 對話腳本生成 | 持續生成多輪對話直到達到所需長度 |
| 資料集生成 | 批量生成訓練資料，每批保持格式與風格一致 |

---

## 使用提示

- 在每次呼叫時，將**前一段輸出的最後幾句**作為接續上下文注入（避免傳入全文以節省 token）
- 設計明確的**生成終止信號**，讓模型或程式知道何時停止
- 維護一份**全局摘要或狀態**（角色、情節、已確立的設定），在每次呼叫時注入
- 使用**明確的接續指令**（「請從以下段落的結尾繼續生成」）避免模型重寫已有內容
- 對於需要一致性的長篇內容，定期進行**全局一致性檢查**

## 提示詞範例

### 範例 1：長篇故事的分批生成

**第一批（初始生成）**
```
請開始寫一篇科幻短篇小說，主題是「一個失憶的 AI 試圖找回自己的記憶」。

全局設定（每批生成都會保持）：
- 主角：ARIA，一個記憶被清除的 AI 助理
- 時間：2087 年，AI 與人類共存的城市
- 風格：緊張、帶有哲學思考，第三人稱視角

請生成約 400 字的第一段，以一個懸念句結尾。
生成結束後，輸出：[CONTINUATION_POINT: 最後一句話]
```

**後續批次（接續生成）**
```
你正在寫一篇科幻短篇小說，請從以下接續點繼續。

全局設定：
- 主角：ARIA，一個記憶被清除的 AI 助理
- 時間：2087 年，AI 與人類共存的城市
- 風格：緊張、帶有哲學思考，第三人稱視角

上一段的最後內容：
[貼入 CONTINUATION_POINT 的內容]

請繼續生成約 400 字，不要重複上述內容，從上一段結尾自然銜接。
生成結束後，再次輸出：[CONTINUATION_POINT: 最後一句話]
輸入「[STORY_END]」表示故事結束。
```

### 範例 2：無限問答集生成

```
請生成一份「Python 進階面試題庫」，每次呼叫生成 5 題。

生成格式（每題）：
---
題目 {n}：[題目標題]
難度：⭐⭐⭐ (1-5星)
問題：[完整問題描述]
標準答案：[詳細解答]
考察重點：[這題在考什麼概念]
---

規則：
- 每次呼叫後，在末尾輸出 [NEXT_INDEX: {下一題的編號}]
- 題目不得與已生成的題目重複（已生成主題：{已生成主題列表}）
- 若輸入 [STOP] 則停止生成

已生成主題：（初次呼叫時為空）
請生成第 1-5 題。
```

### 範例 3：串流式日誌分析報告

```
你正在生成一份系統事件分析報告，新的事件日誌會持續傳入。

報告狀態：
- 已分析事件數：{n}
- 目前識別的問題：{已識別問題清單}
- 報告摘要到目前為止：{當前摘要}

新傳入的日誌片段：
{new_log_entries}

請：
1. 分析新傳入的日誌，識別新的異常或模式
2. 如果有新發現，更新「已識別問題」清單
3. 輸出本批日誌的分析結論（3-5 句）
4. 更新報告摘要（保持在 100 字以內）
5. 輸出更新後的報告狀態（格式同上方「報告狀態」）

若收到 [GENERATE_FINAL_REPORT] 指令，則基於所有已收集資訊生成完整報告。
```

## 常見錯誤

- **未提供接續上下文**：模型不知道前面寫了什麼，導致重複或矛盾的內容
- **接續上下文太長**：將完整已生成內容都注入，導致 context 膨脹
- **缺乏全局狀態追蹤**：長篇生成中角色設定、已確立的事實前後矛盾
- **未設計終止條件**：生成可以無限繼續但沒有明確的結束機制
- **每批生成太長**：單次生成內容過長，品質在後半段明顯下降
