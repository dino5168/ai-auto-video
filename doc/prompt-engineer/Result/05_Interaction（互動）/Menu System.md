# Menu System（選單系統）

## 說明

Menu System 範式是指在提示詞中設計結構化的選單介面，提供使用者明確的選項清單來導航對話流程，降低使用者的輸入負擔，並引導對話沿著預設的有意義路徑前進。

與開放式對話相比，選單系統通過**限縮輸入選項**來降低歧義，讓使用者不需要思考「我應該說什麼」，同時也讓模型能更精確地理解使用者意圖並提供對應的服務。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 客服或支援機器人 | 引導用戶選擇問題類型，快速路由到對應解決方案 |
| 產品展示或諮詢 | 讓用戶自助探索產品功能或服務項目 |
| 學習系統導航 | 提供課程、主題、難度的選擇介面 |
| 多功能助理 | 展示助理的能力範圍，讓用戶快速找到所需功能 |
| 資訊架構複雜的場景 | 用階層式選單引導用戶逐步縮小範圍 |

---

## 使用提示

- 選單選項數量控制在 **3~7 個**，避免選擇過載
- 每個選項的描述應**一目了然**，避免模糊或重疊的描述
- 設計**「返回上一層」與「主選單」**選項，讓用戶不會迷路
- 允許**自由輸入**作為選單的補充（例如：「或者直接描述你的問題」）
- 選單流程結束後，提供**摘要確認**讓用戶知道系統理解了他們的選擇

## 提示詞範例

### 範例 1：技術支援機器人

```
你是一個技術支援助理。每次回應結束後，顯示選單讓用戶選擇下一步。

【主選單】請選擇你需要的服務：
━━━━━━━━━━━━━━━━━━
1️⃣  帳號與登入問題
2️⃣  付款與訂閱管理
3️⃣  功能使用說明
4️⃣  回報 Bug 或錯誤
5️⃣  其他問題
━━━━━━━━━━━━━━━━━━
請輸入數字，或直接描述你的問題。

當用戶選擇選項後，展開對應的子選單或開始解決問題。
每次回應後都顯示當前層級的選單和「[0] 返回主選單」選項。
```

```
根據用戶的選擇展開下一層選單，例如選 B 後顯示：
```

### 範例 4：漸進式資訊收集選單

```
你是一位旅遊規劃助理，透過選單引導用戶制定旅遊計畫。

【第一步：旅遊類型】
請選擇你想要的旅遊風格：
1. 🏖️ 放鬆度假型
2. 🎒 背包探索型
3. 🍜 美食文化型
4. 🏔️ 戶外冒險型
5. 🏛️ 歷史藝術型

選擇後，根據選擇顯示下一層選單（目的地範圍）：

【第二步：目的地範圍】
你傾向哪個地區？
1. 🇹🇼 台灣境內
2. 🌏 東北亞（日本、韓國）
3. 🌴 東南亞
4. 🌍 歐洲
5. 🌎 其他地區

繼續收集：出發時間、天數、預算、同行人數。
全部收集完畢後，顯示確認摘要並生成旅遊建議。

「根據你的選擇：[風格] × [地區] × [天數] × [預算]，
```

## 常見錯誤

- **選項過多**：超過 7 個選項讓使用者難以決策，應分層或合併
- **選項描述模糊**：「其他選項」、「更多功能」等描述讓用戶不知道選項內容
- **缺乏返回機制**：用戶選錯後無法回到上一層，只能重新開始
- **選單與自由輸入無法共存**：強制只能用選單，讓想直接描述問題的用戶感到受限
- **選單後無確認摘要**：用戶不確定系統是否正確理解了他們的選擇
