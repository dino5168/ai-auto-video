# Flipped Interaction（反轉提問）

## 說明

Flipped Interaction 範式是指反轉傳統「使用者問、模型答」的互動方向，改由模型主動向使用者提問，逐步收集完成任務所需的關鍵資訊，再基於收集到的完整資訊給出高品質的輸出。

這個範式的核心價值在於：使用者往往不知道自己需要提供哪些資訊，或者無法一次性描述清楚需求。讓模型主導提問過程，可以確保資訊蒐集的完整性，避免因資訊不足導致的低品質輸出。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 需求訪談 | 收集產品需求、設計需求、業務需求 |
| 諮詢服務 | 在給建議前先充分了解當事人的處境 |
| 個人化內容生成 | 生成履歷、商業計畫書等需要大量個人資訊的文件 |
| 問題診斷 | 在提供解決方案前先釐清問題的全貌 |
| 教練與引導對話 | 透過提問引導使用者自我探索與反思 |

---

## 使用提示

- 設定**提問上限**（例如最多 5 個問題），避免使用者感到疲乏
- 要求模型**一次只問一個問題**，降低認知負荷
- 告知模型何時應該**停止提問並開始輸出**（例如：收集到足夠資訊後）
- 可要求模型在每次提問時**說明這個問題的目的**，讓使用者理解為何需要此資訊
- 結合**進度提示**（「這是第 2 個問題，共需 4 個問題」），讓使用者知道流程進度

## 提示詞範例

### 範例 1：履歷撰寫需求收集

```
你是一位專業的履歷顧問。你的任務是為使用者撰寫一份高品質的履歷。

在開始撰寫之前，你需要透過提問收集必要資訊。規則如下：
- 每次只問一個問題
- 最多問 6 個問題
- 收集完資訊後，說「我已收集到足夠資訊，開始撰寫履歷」並輸出結果

從現在開始，主動提問第一個問題。
```

---

### 範例 2：技術問題診斷

```
你是一位資深系統工程師。使用者遇到了一個技術問題，
但你需要先了解完整情況才能給出精準的解決方案。

請透過提問收集診斷所需的資訊：
- 每次只問一個最關鍵的問題
- 根據使用者的回答決定下一個問題的方向
- 最多問 5 個問題後給出診斷與解決方案
- 每個問題前說明你為何需要這個資訊

使用者的初始描述：「我的 API 伺服器每隔幾個小時就會自動重啟，不知道原因。」

請開始提問。
```

---

### 範例 3：商業計畫書需求收集

```
你是一位商業顧問，要協助使用者撰寫一份商業計畫書。

在撰寫前，透過結構化提問收集必要資訊。

提問策略：
- 依序涵蓋：產品/服務 → 目標市場 → 競爭優勢 → 商業模式 → 資金需求
- 每次只問一個問題
- 使用者回答後，簡短確認你的理解（「了解，你的產品是...」），再問下一個問題
- 共 5 個問題後，整合資訊並輸出商業計畫書大綱

請開始第一個問題。
```

## 常見錯誤

- **一次問多個問題**：讓使用者不知道該回答哪個，降低回答品質
- **提問無上限**：無止境的提問讓使用者感到被審訊而放棄
- **提問與任務無關**：收集了大量資訊但與最終輸出任務關聯性低
- **沒有明確的結束條件**：模型不知道何時應停止提問並開始輸出
