# Context Injection（上下文注入）

## 說明

Context Injection 範式是指將外部資訊（文件內容、資料庫查詢結果、API 回傳值、工具執行結果等）動態地注入到 prompt 中，讓模型能夠基於當下最新、最相關的資訊進行推理與回答，而非僅依賴訓練時固化的知識。

這個範式是 LLM 與外部世界連接的基礎機制——透過注入即時資料，可以讓模型處理訓練截止後的新資訊、私有資料、以及使用者的個人化上下文。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 即時資料查詢 | 注入當前股價、天氣、新聞等即時資訊 |
| 私有知識庫問答 | 注入企業內部文件、規範、FAQ 等模型沒有訓練到的資料 |
| 個人化應用 | 注入用戶的個人資料、偏好、歷史行為 |
| 工具呼叫結果整合 | 將程式執行結果、搜尋結果、計算結果注入後請模型解讀 |
| 動態業務邏輯 | 注入當前的商品庫存、訂單狀態、系統配置等 |

---

## 使用提示

- 使用明確的結構標籤（`<document>`、`<data>`、`<tool_result>`）區分注入內容與問題本身
- 注入前先**過濾與壓縮**：只注入與當前問題相關的部分
- 明確告知模型注入資料的**來源與時效性**（「以下資料來自今日 09:00 的系統快照」）
- 當注入多份文件時，為每份文件加上**識別標籤**方便模型引用
- 指示模型**優先使用注入資料**，而非依賴訓練知識（適用於私有或最新資訊場景）

### 範例 1：企業內部政策問答

```
請根據以下公司政策文件回答員工的問題。
若答案在文件中找不到，請明確說明「此問題需向 HR 確認」，不要自行推測。

<policy_document source="員工手冊 v2.3, 2025-01-15">
第四章 請假規定
4.1 年假：到職滿一年者享有 10 天年假，每增加一年增加 1 天，最高 30 天。
4.2 病假：每年 30 天，前 30 天全薪，需附醫療證明。
4.3 特休：婚假 8 天、喪假視親等 3-8 天。
4.4 請假流程：需提前 3 個工作天申請，緊急狀況事後 24 小時內補申請。
</policy_document>

員工問題：我到職剛好兩年，請問我今年有幾天年假？如果我突然生病沒辦法提前申請怎麼辦？
```

## 常見錯誤

- **注入與問題無關的資訊**：增加 context 負擔，稀釋模型的注意力
- **未標示資料來源與時效**：模型無法判斷資料的可信度與時效性
- **未告知模型優先使用注入資料**：模型可能混用訓練知識與注入資料，產生矛盾
- **注入格式不一致**：不同資料使用不同格式混雜，增加模型解析難度
