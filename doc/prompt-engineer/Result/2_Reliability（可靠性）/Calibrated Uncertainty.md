# Calibrated Uncertainty（校準不確定性）

## 說明

Calibrated Uncertainty 範式是指要求模型對自己的輸出明確表達信心水準，標示不確定的邊界，並在信心不足時主動提示使用者——而非以同樣自信的語氣陳述所有內容，不論其可靠程度。

「校準」（Calibrated）的意思是：模型表達的信心程度應與實際準確率相符。如果模型說「我有 90% 的把握」，那在類似情境中它應該有 90% 的機率是對的。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 知識邊界探索 | 詢問模型訓練資料可能不足的領域 |
| 最新資訊查詢 | 涉及模型訓練截止後可能已更新的資訊 |
| 高風險決策支援 | 醫療、法律、財務等領域不容許錯誤陳述 |
| 研究資料整合 | 需要區分已確立知識與仍有爭議的研究 |
| 建立使用者信任 | 讓使用者知道何時應該自行驗證 |

---

## 使用提示

- 定義信心等級的尺度（高/中/低，或百分比，或文字描述）
- 要求模型對每個重要陳述都標示信心等級
- 說明信心低時模型應如何處理（提示查證，而非拒絕回答）
- 可要求模型說明信心等級背後的依據（為何有把握或為何不確定）
- 避免要求模型「只說確定的事」——這會讓模型過度保守或拒絕提供有用資訊

## 提示詞範例

### 範例 1：信心等級標示

```
請回答以下問題，並對每個主要陳述標示信心等級：
- 🟢 高信心：我非常確定這是正確的
- 🟡 中信心：我認為這是正確的，但建議查證
- 🔴 低信心：我不太確定，請務必查證原始資料

問題：Python 的 GIL（全域解釋器鎖）對多執行緒程式的影響為何？
Python 3.12 有什麼重要的效能改進？
```

---

### 範例 2：百分比信心表達

```
請回答以下醫療相關問題。對於每個陳述，請在句尾標示你的信心百分比，例如 [信心: 85%]。
信心低於 60% 的陳述，請加上「建議諮詢醫療專業人員確認」。

問題：益生菌補充劑對腸躁症（IBS）的治療效果如何？
目前有哪些主流的治療方式？
```

---

### 範例 3：不確定邊界明確化

```
請回答以下問題，並在回答結尾加上「不確定性摘要」區塊，列出：
1. 你最不確定的 2~3 個陳述
2. 這些不確定性的原因（訓練資料不足、資訊時效性、領域爭議等）
3. 使用者應該去哪裡查證（建議查詢的資料來源類型）

問題：量子運算目前在密碼學領域的威脅程度如何？主要的防禦方向是什麼？
```

---

### 範例 4：爭議性話題的信心校準

```
針對以下話題，請整理目前的研究共識，並明確區分：
- **已有強烈共識**（多數研究結論一致）
- **仍有爭議**（研究結果不一致或方法論有爭議）
- **尚無足夠研究**（研究數量不足以做出結論）

話題：間歇性斷食對代謝健康的長期影響。
```

## 常見錯誤

- **信心等級不分化**：模型對所有陳述都標示「高信心」，失去校準的意義
- **過度保守**：要求太嚴導致模型對所有內容都說「不確定」，失去實用價值
- **信心與準確率脫鉤**：模型說「高信心」但實際上是錯的——這是最危險的情況，需要用其他範式（如 Fact Check）輔助
- **忽略說明不確定原因**：只標示等級但不說明為何不確定，使用者無法判斷如何查證
