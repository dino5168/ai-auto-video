# Self-Consistency（自我一致性）

## 說明

Self-Consistency 範式是指對同一個問題進行多次獨立採樣，讓模型從不同推理路徑出發得到答案，最後透過「多數決」或「取交集」的方式選出最穩定可靠的答案。

這個範式基於一個觀察：模型的推理過程具有隨機性，單次輸出可能走入錯誤的推理路徑；但多條不同路徑如果收斂到同一個答案，這個答案的可靠性就大幅提升。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 高精度推理問題 | 數學、邏輯、程式除錯等需要正確答案的任務 |
| 重要決策建議 | 希望確認建議的穩定性，而非單一觀點 |
| 分類或評估任務 | 確認分類結果的一致性 |
| 降低幻覺風險 | 交叉驗證多次輸出，找出穩定的事實陳述 |
| 評估模型信心 | 透過一致性程度間接衡量模型對問題的把握程度 |

---

## 使用提示

- 在單一 prompt 中，可要求模型「用三種不同方法解題，最後比較答案」
- 明確要求每次推理路徑獨立，不要相互參照
- 多數決適用於有明確答案的問題；交集法適用於需要提取穩定共識的問題
- 若三次結果不一致，這本身就是一個有價值的訊號——說明問題的難度或模型的不確定性

## 提示詞範例

### 範例 1：單 Prompt 多路徑數學解題

```
請用三種完全不同的解題方法解以下問題，每種方法獨立進行，不要互相參照。
最後比較三個答案，若一致則確認答案，若不一致則找出哪個方法有誤。

問題：
一個長方形的周長是 48 公分，長是寬的 1.5 倍，請問這個長方形的面積是多少？

方法一：用代數方程式解
方法二：用嘗試法（列出可能的組合）
方法三：用幾何關係圖形推導

最後：
- 三個方法的答案為何？
- 是否一致？
- 最終確認答案為何？
```

---

### 範例 2：多角度分析取交集

```
請從三個完全不同的分析角度，獨立評估以下商業決策的風險：

決策：將公司的主要資料庫從自建機房遷移至公有雲（AWS）。

角度一：資安與合規風險
角度二：營運與可用性風險
角度三：財務與成本風險

每個角度獨立列出前 3 大風險。

最後：
整合三個角度，找出「被多個角度同時提及」的核心風險，這些是優先需要處理的問題。
```

---

### 範例 3：分類一致性驗證

```
請對以下客戶評論進行情感分類，用三種不同的判斷框架各自獨立分類一次：

框架一：以整體情感傾向判斷（正面/負面/中性）
框架二：以客戶是否會回購的意願判斷（高意願/低意願/不確定）
框架三：以客戶是否會推薦給他人判斷（會推薦/不會推薦/不確定）

評論：「東西很快就到了，品質還可以，但包裝有點簡陋，希望下次能改善。」

完成三個框架的分類後，說明：
- 三個框架的結論是否一致？
- 若有分歧，原因為何？
- 綜合判斷這是一個「正面」還是「需要關注」的評論？
```

## 常見錯誤

- **路徑未真正獨立**：模型第二次解題時受到第一次答案的影響，無法達到真正獨立採樣
- **多數決用於開放性問題**：Self-Consistency 適合有確定答案的問題，不適合創意或主觀性任務
- **忽略不一致的訊號**：三次答案不同時直接取其一，而沒有深入分析不一致的原因
- **採樣次數不足**：兩次採樣在結果不同時無法決定多數，建議至少三次
