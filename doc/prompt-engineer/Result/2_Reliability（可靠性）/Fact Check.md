# Fact Check（事實查核）

## 說明

Fact Check 範式是指在提示詞中要求模型主動標示不確定的資訊、區分事實與推論、並在可能的情況下引用來源或說明資訊依據。

這個範式的目的是減少模型的「幻覺」（Hallucination）問題，讓使用者能夠清楚辨識哪些內容是可信的、哪些需要進一步查證。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 高風險決策支援 | 醫療、法律、財務等領域，錯誤資訊代價高昂 |
| 知識型問答 | 詢問模型具體的事實、數據、歷史事件等 |
| 研究資料整理 | 需要區分已確認資訊與待驗證資訊 |
| 新聞或報告摘要 | 防止模型在摘要過程中捏造細節 |
| 技術文件核查 | 確認技術規格、API 用法等是否有依據 |

---

## 使用提示

- 明確要求模型標示不確定項目（例如用 `[待查證]` 標記）
- 要求模型區分「直接知識」與「推論」
- 可要求模型說明資訊的大概時效性（訓練資料截止日期的限制）
- 搭配要求「若不確定請直接說不知道」，避免模型強行填答

---

## 提示詞範例

### 範例 1：基本事實查核標示

```
請回答以下問題，並遵守以下規則：
- 你確定的事實：直接陳述
- 你不確定的資訊：在句尾加上 [待查證]
- 你完全不知道的內容：明確說明「我沒有這方面的資料」
- 若資訊有時效性限制：說明「此資訊截至我的訓練資料，可能已有更新」

問題：台灣目前的法定最低工資是多少？相關法規依據為何？
```

---

### 範例 2：事實與推論分離

```
請分析以下市場情況，並在回答中明確區分：
- **確認事實**（有明確依據的資訊）
- **合理推論**（基於事實的邏輯推導）
- **不確定項目**（需要進一步查證的部分）

每個段落請在開頭標明類型，例如：
[事實] 特斯拉於 2023 年的全球交付量為...
[推論] 基於上述數據，可推測...
[不確定] 關於其在台灣市場的具體份額...

請分析：電動車市場在亞洲的發展現況與趨勢。
```

---

### 範例 3：醫療諮詢場景的謹慎標示

```
請回答以下健康相關問題，並嚴格遵守以下標準：
1. 對於有醫學共識的內容，可以直接說明
2. 對於有爭議或研究結論不一致的內容，說明「目前研究結果不一致」
3. 所有建議結尾加上「建議諮詢專業醫師確認」
4. 不要給出具體的藥物劑量或診斷結論

問題：間歇性斷食對第二型糖尿病患者有幫助嗎？
```

## 常見錯誤

- **未要求標示不確定性**：模型預設會以自信語氣回答，不主動說明限制
- **要求過於嚴苛**：要求模型「只說有 100% 把握的事」會讓輸出過於保守甚至拒絕回答
- **忽略時效性問題**：未提醒模型說明訓練資料截止日期的限制
- **事實與推論混淆**：未要求模型區分兩者，讀者難以判斷哪些需要驗證
