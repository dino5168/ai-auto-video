# Memory（記憶管理）

## 說明

Memory 範式是指讓 LLM 應用能夠跨對話會話保留重要資訊，主要分為兩種形式：

- **短期工作記憶（In-Context Memory）**：將需要保留的資訊直接存放在當前 context window 中，僅在當次會話有效
- **長期外部記憶（External Memory）**：將重要資訊持久化儲存到外部系統（資料庫、向量庫、文件），在新的會話開始時重新注入

Memory 範式解決了 LLM「失憶」的根本問題——每次新的對話預設從零開始，而 Memory 讓應用能夠記住用戶偏好、對話歷史、任務狀態等跨越會話的關鍵資訊。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 個人助理應用 | 記住用戶的偏好、習慣、背景資訊 |
| 長期專案協作 | 保留專案脈絡、已決定事項、待辦清單 |
| 客服系統 | 記住客戶的歷史問題、已解決的工單、偏好設定 |
| 學習輔助工具 | 追蹤學習進度、記住已掌握與尚未學習的概念 |
| Multi-Agent 系統 | Agent 之間共享工作狀態與中間結果 |

---

## 使用提示

- 明確區分**值得記憶的資訊**（用戶偏好、重要決策）與**不需要記憶的資訊**（臨時計算結果）
- 對長期記憶進行**定期清理與更新**，避免過時資訊干擾
- 記憶注入時，標注**記錄時間**以便模型判斷資訊的時效性
- 設計記憶的**結構化格式**，方便程式解析、更新與查詢
- 短期記憶可使用**滾動摘要**策略，避免 context 膨脹

## 提示詞範例

### 範例 1：短期工作記憶（會話內記憶更新）

```
你是一個程式開發助手。請在每次回答後，更新以下工作記憶區塊，
記錄我們在本次會話中確認的重要決策。

<working_memory>
專案：電商後台系統
已確認決策：
- 語言：Python (FastAPI)
- 資料庫：PostgreSQL
- 部署：Docker + AWS ECS
待討論：認證方案（OAuth2 或 JWT）
</working_memory>

問題：我想了解 OAuth2 和 JWT 的差異，幫助我決定認證方案。

（回答完問題後，請更新 working_memory 中的「待討論」項目）
```

### 範例 2：長期記憶注入（跨會話）

```
以下是根據你過去的使用記錄建立的個人化記憶，請在本次對話中參考這些資訊。

<user_memory last_updated="2026-04-10">
用戶背景：
- 職位：後端工程師，5 年 Python 經驗
- 目前學習目標：學習 Rust 語言
- 已完成：Rust 基礎語法、所有權概念
- 卡關點：生命週期標注（lifetime annotations）

偏好設定：
- 解釋風格：喜歡用 Python 做類比
- 回答長度：喜歡簡潔的解釋 + 一個實際範例
- 不喜歡：過多的理論背景，希望直接看到程式碼

上次對話摘要（2026-04-10）：
討論了 Rust 的 borrow checker 機制，理解了不可變借用與可變借用的規則。
</user_memory>

問題：生命週期標注到底是在做什麼？我完全搞不懂。
```

### 範例 3：結構化記憶的更新指令

```
你正在管理一個客戶服務案例。請根據本次對話更新客戶記憶檔案。

<customer_memory customer_id="C-10234">
姓名：陳小華
問題歷史：
- 2026-03-15：詢問退貨政策 → 已解決
- 2026-04-01：產品功能問題 → 已解決
目前未解決問題：無
VIP 等級：Silver
特殊備註：對等待時間敏感，偏好文字溝通
</customer_memory>

本次對話內容：
客戶反映收到的商品顏色與網站圖片不符，要求換貨。
已確認：確實為攝影燈光導致色差，但客戶主觀認為差異過大。
處理結果：同意免費換貨，已建立換貨單號 EX-20260414-007。

請輸出更新後的完整 customer_memory XML，反映本次對話的處理結果。
```

### 範例 4：Multi-Agent 共享狀態記憶

```
你是資料分析 Agent。以下是目前任務的共享狀態，
由協調 Agent 維護，所有 Agent 都需要讀取並在完成工作後更新。

<shared_task_state task_id="ANALYSIS-2026-042">
任務：分析 Q1 2026 銷售數據並生成報告
總體進度：40%

子任務狀態：
- [完成] 數據清洗 (data_agent) → 輸出路徑: /data/cleaned/q1_2026.csv
- [進行中] 趨勢分析 (analysis_agent = 你) → 開始時間: 14:30
- [待開始] 視覺化圖表 (viz_agent) → 等待分析結果
- [待開始] 報告撰寫 (writing_agent) → 等待圖表完成

你的輸入：/data/cleaned/q1_2026.csv
你的輸出路徑：/data/analysis/q1_2026_trends.json
</shared_task_state>

請執行趨勢分析，完成後輸出你需要更新的 shared_task_state 欄位內容。
```

## 常見錯誤

- **記憶未結構化**：非結構化的記憶難以程式化讀取、更新與查詢
- **過時記憶未清理**：舊的偏好或狀態資訊干擾當前的推理
- **記憶容量無限增長**：長期記憶沒有清理策略，最終超出 context 限制
- **記憶與即時輸入衝突未處理**：記憶說用戶喜歡 A，但用戶當下要求 B，未明確定義優先順序
