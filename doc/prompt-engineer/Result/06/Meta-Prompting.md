# Meta-Prompting（元提示詞）

## 說明

Meta-Prompting 範式是指讓模型自行生成、評估或最佳化 prompt，實現「prompt 生成 prompt」的自動化機制。在這個範式中，模型不再只是執行任務的工具，而是成為設計執行工具本身的工程師。

這個範式是自動化 prompt 工程的基礎，代表性框架包括 DSPy（Declarative Self-improving Python）——它讓系統能夠根據任務目標自動推導出最佳的 prompt 結構，而非依賴人工手動調整。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| Prompt 自動最佳化 | 基於任務目標自動生成並測試不同 prompt 版本 |
| 任務適配 | 將通用 prompt 模板自動調整為特定任務的最佳版本 |
| Prompt 品質評估 | 讓模型評估現有 prompt 的優缺點並提出改進版本 |
| 快速 Prompt 原型設計 | 給定任務描述，自動生成候選 prompt 供人工選擇 |
| 自適應 Agent | Agent 能夠根據執行結果自動調整自身的指令 |

---

## 使用提示

- 明確定義 prompt 的**評估標準**（準確性、格式一致性、完整性等可量化指標）
- 要求模型在生成 prompt 時**說明設計決策**（為何使用這個結構、這個措辭）
- 設計**反饋迴圈**：讓模型執行自生成的 prompt，根據結果再改進
- 指定 prompt 需要符合的**約束條件**（長度上限、必須包含的元素、禁止的模式）
- 考慮使用**Few-Shot 示例**來校準模型對「好 prompt」的理解標準

## 提示詞範例

### 範例 1：任務導向的 Prompt 生成

```
你是一位提示詞工程師。給定一個任務描述，你需要生成一個高品質的 prompt。

評估一個好 prompt 的標準：
- 角色定義清晰（模型知道自己是誰）
- 任務說明具體（模型知道要做什麼）
- 輸出格式明確（模型知道怎麼呈現結果）
- 包含必要的約束條件
- 有適當的示例（如果需要）

任務描述：
「我需要一個 prompt，讓模型幫助用戶診斷他們 Python 程式中的效能問題。
用戶會貼上程式碼，模型應該找出瓶頸並提供優化建議。
目標用戶是中級 Python 開發者。」

請生成：
1. 設計思路（你考慮了哪些因素？）
2. 完整的 prompt
3. 這個 prompt 的潛在弱點與改進空間
```

### 範例 2：Prompt 評估與改進

```
你是一位提示詞審查專家。請評估以下 prompt，並提供改進版本。

評估維度（各 25 分，滿分 100）：
- 清晰度：指令是否明確、無歧義
- 完整性：是否涵蓋所有必要資訊
- 效率：是否有冗餘或可精簡的部分
- 穩健性：面對邊界情況是否有足夠指引

待評估的 prompt：
「請幫我寫一篇關於 AI 的文章，要有趣，大概 500 字，給一般人看的。」

請輸出：
1. 各維度評分與評語
2. 主要問題列表
3. 改進後的 prompt 版本
4. 改進前後的關鍵差異說明
```

### 範例 3：自動 Prompt 最佳化迴圈

```
你將執行一個 prompt 最佳化迴圈，共進行 3 輪迭代。

目標任務：讓模型從一段新聞中提取：事件、時間、地點、涉及人物、影響。

評估方法：對以下測試案例運行 prompt，檢查輸出是否符合預期格式與完整性。

測試案例：
「台積電昨日宣布，將在高雄興建第三座 2 奈米晶圓廠，預計投資額達新台幣 1.5 兆元，
並將創造約 8,000 個高科技就業機會，預計 2029 年開始量產。
經濟部長林○○表示，此舉將強化台灣半導體供應鏈韌性。」

迴圈執行：
輪次 1：
  a. 生成初版 prompt
  b. 模擬以測試案例執行該 prompt，輸出結果
  c. 評估結果（哪些資訊提取正確？哪些遺漏或錯誤？）

輪次 2：
  a. 根據輪次 1 的評估改進 prompt
  b. 再次模擬執行，比較與輪次 1 的差異

輪次 3：
  a. 進一步改進
  b. 輸出最終版本 prompt 與三輪迭代的改進摘要

開始執行迴圈。
```

### 範例 4：自適應 Agent 的 Prompt 自我修正

```
你是一個自適應 Agent，能夠根據執行結果修改自身的操作 prompt。

你當前的操作 prompt（版本 1.0）：
「請分析用戶輸入的文字，識別其中的主要情緒，回傳 positive/negative/neutral。」

以下是最近 5 次執行的結果與人工標注的正確答案：

| 輸入摘要 | 模型輸出 | 正確答案 | 是否正確 |
|----------|----------|----------|----------|
| 「產品很好但服務差」 | positive | mixed | ❌ |
| 「還可以，沒什麼特別」 | neutral | neutral | ✅ |
| 「完全不推薦！」 | negative | negative | ✅ |
| 「雖然貴但值得」 | negative | positive | ❌ |
| 「功能夠用，不會再買」 | neutral | negative | ❌ |

請：
1. 分析失敗案例的共同模式
2. 識別當前 prompt 的根本缺陷
3. 生成改進後的 prompt（版本 1.1）
4. 解釋修改如何解決識別到的缺陷
5. 預測版本 1.1 在以上 5 個案例的表現
```

## 常見錯誤

- **評估標準不明確**：沒有定義「好 prompt」的具體標準，導致模型生成的 prompt 缺乏針對性
- **缺乏反饋迴圈**：只生成 prompt 但不執行驗證，無法知道生成的 prompt 是否真的更好
- **過度依賴 Meta-Prompting**：把所有 prompt 設計都外包給模型，忽略了人工領域知識的價值
- **迭代次數無上限**：沒有設定最佳化的終止條件，可能無限迭代
- **測試案例不具代表性**：最佳化的 prompt 只在測試案例上好，在真實分佈上反而更差（過擬合）
