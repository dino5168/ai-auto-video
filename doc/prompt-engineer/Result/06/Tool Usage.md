# Tool Usage（工具呼叫）

## 說明

Tool Usage 範式是指讓模型在推理過程中識別何時需要外部能力，並透過定義好的介面呼叫外部工具——包括搜尋引擎、程式碼執行環境、API 端點、資料庫查詢、計算器等——再將工具回傳的結果整合入後續的推理與輸出。

這個範式打破了 LLM 純文字生成的侷限，讓模型能夠存取即時資訊、執行精確計算、操作外部系統，從而完成僅靠語言生成無法達成的任務。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 即時資訊查詢 | 搜尋最新新聞、股價、天氣等訓練資料截止後的資訊 |
| 精確數值計算 | 避免模型直接計算造成的數學錯誤，呼叫計算器工具 |
| 程式碼執行驗證 | 讓模型撰寫程式後實際執行，驗證輸出是否正確 |
| 資料庫查詢 | 從結構化資料庫取得精確數據，而非依賴模型記憶 |
| 文件操作 | 讀取、寫入、搜尋本地或雲端文件系統 |

---

## 使用提示

- 在 prompt 中**明確定義每個可用工具的名稱、功能、輸入參數與輸出格式**
- 要求模型在呼叫工具前**說明呼叫理由**（我需要查詢 X，因為 Y）
- 設計**工具呼叫失敗的處理邏輯**（工具無回應或回傳錯誤時的備案）
- 工具回傳值注入後，要求模型**明確引用工具結果**而非自行推測
- 限制每輪對話的**工具呼叫次數上限**，避免無限迴圈

## 提示詞範例

### 範例 1：定義工具介面（Function Calling 風格）

```
你是一個研究助理，可以使用以下工具：

工具清單：
1. web_search(query: string) → {results: [{title, url, snippet}]}
   用途：搜尋網路上的即時資訊
   
2. calculate(expression: string) → {result: number, error?: string}
   用途：執行數學計算，避免自行計算造成錯誤
   
3. get_stock_price(symbol: string) → {price: number, change: number, timestamp: string}
   用途：取得股票即時價格

使用工具時，請用以下格式：
<tool_call>
{"tool": "工具名稱", "params": {"參數名": "參數值"}, "reason": "呼叫原因"}
</tool_call>

工具回傳結果會以 <tool_result> 標籤提供，請根據結果繼續回答。

問題：台積電（TSM）今天的股價是多少？如果我持有 500 股，目前市值是多少？
```

---

### 範例 2：程式碼執行驗證迴圈

```
你是一個 Python 程式設計助理，可以使用 code_executor 工具驗證你撰寫的程式碼。

工具：
- code_executor(code: string, language: "python") → {stdout: string, stderr: string, exit_code: number}

工作流程：
1. 撰寫程式碼
2. 呼叫 code_executor 執行
3. 若有錯誤（exit_code != 0），根據 stderr 修正並重新執行
4. 確認輸出正確後，向用戶展示最終程式碼與執行結果

任務：請寫一個函式，找出一個整數列表中的所有質數，並用 [2, 3, 4, 5, 17, 20, 23] 測試。
```

---

### 範例 3：多工具協作查詢

```
你是一個商業分析助理，有以下工具可用：

- database_query(sql: string) → {rows: object[], row_count: number}
  資料庫包含 orders 表（order_id, customer_id, amount, date, status）
  
- calculate(expression: string) → {result: number}

- generate_chart_data(data: object[], chart_type: string) → {chart_config: object}
  用於生成圖表設定

請回答以下問題，每次使用工具前說明你的推理，工具回傳後整合結果再繼續。

問題：請分析今年（2026年）Q1 各月份的訂單總金額趨勢，並找出哪個月份的平均訂單金額最高。
```

## 常見錯誤

- **工具介面定義模糊**：參數型別、回傳格式不明確，導致模型呼叫方式不一致
- **模型自行「假裝」工具已執行**：沒有確保模型等待真實工具回傳，而是自行捏造結果
- **缺乏工具失敗處理**：工具呼叫失敗時模型無所適從，未定義備案行為
- **無限工具呼叫迴圈**：未設置呼叫次數上限，模型陷入反覆呼叫的迴圈
- **工具結果未被引用**：模型獲得工具結果後，在最終回答中仍使用自身推測而非工具數據
