# Game Play（遊戲化互動）

## 說明

Game Play 範式是指在提示詞中設計一套遊戲化的互動框架，包括角色扮演情境、謎題機制、規則系統、積分/進度追蹤等遊戲元素，讓與模型的互動從單純的問答轉變為具有參與感與沉浸感的體驗。

遊戲化的核心價值在於：透過明確的規則、目標與即時回饋，提升使用者的投入度與學習動機，同時讓複雜的互動流程更直觀易懂。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 語言學習 | 透過角色扮演對話或猜詞遊戲練習語言技能 |
| 知識測驗 | 設計問答競賽、闖關測驗等互動學習形式 |
| 技能培訓 | 情境模擬訓練（銷售演練、面試練習、危機處理） |
| 創意寫作 | 合作敘事、選擇你的冒險故事、世界觀建構 |
| 使用者參與度提升 | 讓原本枯燥的任務變得有趣 |

---

## 使用提示

- 在開始時**完整說明遊戲規則**，包括目標、勝負條件、可用操作
- 設計**明確的狀態追蹤**（生命值、積分、進度、關卡）
- 模型應在每輪互動後**更新並顯示當前遊戲狀態**
- 設定**合理的難度曲線**，讓遊戲有挑戰性但不讓人沮喪
- 為不同結果設計**有意義的回饋**，而非單純的「正確/錯誤」

## 提示詞範例

### 範例 1：英語單字猜謎遊戲

```
我們來玩一個英語單字猜謎遊戲！

遊戲規則：
- 你心中想一個英語單字（5個字母）
- 我每次猜一個單字，你告訴我：
  🟩 = 字母正確且位置正確
  🟨 = 字母存在但位置錯誤
  ⬛ = 字母不存在
- 我有 6 次機會猜對
- 每次回應請顯示：本次結果 + 已猜字母清單 + 剩餘次數

遊戲狀態顯示格式：
```
第 X 次猜測
[單字]: [🟩🟨⬛⬛🟩]
已排除字母: [a, b, c...]
剩餘次數: X
```

請先告訴我你已想好一個單字（不要說出來），然後我開始猜。
```

### 範例 2：程式設計學習闖關

```
歡迎來到「Python 冒險」！你將扮演一位剛入門的巫師學徒，
透過解決程式謎題來提升等級。

角色狀態：
- 等級：1（見習學徒）
- 經驗值：0 / 100
- 已解鎖技能：變數、基本運算

闖關規則：
- 每關提供一個程式問題，你需要寫出 Python 程式碼
- 答對得 20 經驗值，答錯可以再試一次（得 10 經驗值）
- 累積 100 經驗值升一級，解鎖新技能
- 使用提示「求助」可獲得一個線索（扣 5 經驗值）

第一關：
「魔法師需要計算三個魔法水晶的總能量值。
請寫一段程式，宣告三個變數分別儲存值 42、17、88，
然後計算並印出它們的總和。」

輸入你的程式碼，或輸入「求助」獲得提示。
```

### 範例 3：銷售情境模擬訓練

```
你即將進行一場銷售技巧模擬訓練。

情境設定：
你是一位 SaaS 產品的銷售代表，正在與潛在客戶進行初次會議。
我將扮演客戶（一家中型製造業的 IT 主管）。

評分系統（每次對話後更新）：
- 需求挖掘：0/25 分
- 產品定位：0/25 分
- 異議處理：0/25 分
- 成交推進：0/25 分
- 總分：0/100 分

遊戲規則：
- 對話進行 10 輪後結束，給出最終評分與建議
- 每輪對話結束後，我會以客戶身份回應，並在回應後用 [教練評語] 標注本輪的優缺點
- 若你表現出色，客戶的「興趣度」會上升（顯示為 😐→🙂→😊→🤝）

目前客戶興趣度：😐

請開始你的開場白。
```

## 常見錯誤

- **規則說明不完整**：遊戲開始後才發現有未定義的情況，打斷沉浸感
- **狀態追蹤不一致**：模型遺忘或計算錯誤遊戲狀態（積分、進度）
- **遊戲與學習目標脫節**：遊戲化形式有趣但沒有達到原本的學習或任務目標
- **難度設計失衡**：太簡單失去挑戰性，太難讓使用者放棄
