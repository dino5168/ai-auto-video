# Simulation / Counterfactual（模擬與反事實推理）

## 說明

Simulation / Counterfactual 範式是指讓模型模擬假設情境或進行反事實推理，探索「如果某個條件改變，結果會如何」的問題空間，用於理解邊界條件、風險來源、決策影響與系統行為。

- **模擬（Simulation）**：構建一個假設的情境模型，推演在該情境下事件的發展過程
- **反事實推理（Counterfactual）**：改變歷史或現狀中的某個關鍵變數，推導出不同的結果

這個範式特別適合在真實世界中無法或不應直接實驗的情況下，進行低成本的思維實驗。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 風險評估 | 模擬最壞情況（Worst Case）發生時的系統行為 |
| 決策後評估 | 反思「如果當初做了不同的決定，結果會不同嗎？」 |
| 邊界條件探索 | 找出系統在什麼條件下會失敗或表現異常 |
| 策略規劃 | 預演競爭對手的可能反應或市場變化 |
| 學習與培訓 | 透過假設情境讓學習者思考不同決策的後果 |

---

## 使用提示

- 明確定義假設的起點（什麼改變了）與不變的條件（什麼保持原樣）
- 要求模型說明推理的邏輯鏈：A 改變 → 導致 B → 進而影響 C
- 可要求模型同時模擬「樂觀情境」、「悲觀情境」與「最可能情境」
- 反事實推理完成後，引導出對現實決策的啟示

## 提示詞範例

### 範例 1：系統故障場景模擬

```
請模擬以下故障場景，推演事件的發展過程與影響範圍：

系統架構：一個電商平台，使用微服務架構，包含：
用戶服務、商品服務、訂單服務、支付服務、通知服務，
以及一個共用的 Redis 快取層。

故障情境：Redis 叢集在週末購物高峰期完全停止回應。

請推演：
1. 哪些服務會立即受到影響？如何受影響？
2. 影響會如何在各服務之間擴散？
3. 最終使用者會看到什麼症狀？
4. 哪些服務可能會觸發 Cascade Failure？
5. 在沒有任何自動恢復機制的情況下，30 分鐘後系統狀態如何？
```

### 範例 2：商業決策反事實分析

```
請進行反事實分析：

現實情況：A 公司在 2020 年選擇了以廣告為主的商業模式，
目前月活用戶 500 萬，年營收 2 億台幣，但用戶增長趨於平緩。

反事實假設：如果 A 公司當初選擇了訂閱制（每月 99 元，無廣告），
而非廣告模式，現在的情況可能是如何？

請分析：
1. 訂閱制對早期用戶增長速度的影響
2. 在相同時間點，訂閱制下的可能用戶數與營收
3. 兩種模式的長期競爭優勢差異
4. 這個反事實分析對 A 公司現在的策略調整有什麼啟示？
```

### 範例 3：多情境並行模擬

```
針對以下商業決策，請同時模擬三種情境的發展：

決策：一家實體零售商決定將 40% 的實體門市轉型為純線上業務。

請模擬：

**樂觀情境**（最有利的外部條件）
- 前提假設是什麼？
- 18 個月後的業務狀態

**悲觀情境**（最不利的外部條件）
- 前提假設是什麼？
- 18 個月後的業務狀態

**最可能情境**（基於現有資料的合理預測）
- 前提假設是什麼？
- 18 個月後的業務狀態

最後：三個情境對比後，這個決策的關鍵風險因子是什麼？
```

### 範例 4：歷史反事實推理

```
請進行以下反事實推理練習：

歷史事件：2008 年金融危機中，美國政府決定讓雷曼兄弟破產。

反事實假設：如果美國政府選擇像救助貝爾斯登一樣，
為雷曼兄弟提供緊急救助，避免其破產，接下來的事件發展可能如何？

請分析：
1. 對當時金融市場恐慌情緒的短期影響
2. 對後續 AIG、花旗等機構的救助決策的影響
3. 對 2009 年全球經濟衰退深度的可能影響
4. 「道德風險」問題是否會以不同形式浮現？
5. 這個反事實對現代金融監管政策設計有什麼啟示？
```

## 常見錯誤

- **假設條件定義不清**：沒有明確說明什麼改變了、什麼保持不變，導致模擬基準不一致
- **推理鏈跳躍**：直接跳到最終結果而沒有說明中間的因果機制
- **忽略二階效應**：只考慮直接影響，沒有追蹤影響如何擴散到其他系統或行為者
- **模擬流於空想**：沒有與現實決策連結，無法從模擬中提取有用的行動啟示
