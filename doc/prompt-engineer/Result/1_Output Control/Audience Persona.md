# Audience Persona（目標受眾設定）

## 說明

Audience Persona 範式是指在提示詞中描述目標受眾的特徵，讓模型根據受眾的背景知識、專業程度、閱讀習慣與需求，自動調整解釋深度、用詞選擇與內容假設前提。

與 Persona 不同之處在於：Persona 是設定「模型的身份」，Audience Persona 是設定「閱讀這份輸出的人是誰」。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 跨專業溝通 | 將技術內容翻譯給非技術受眾理解 |
| 教育內容設計 | 針對不同年齡層或學習階段調整說明方式 |
| 行銷文案撰寫 | 根據目標客群的興趣與痛點調整訴求 |
| 文件撰寫 | 針對初學者或專家分別產生不同深度的說明文件 |

---

## 使用提示

- 具體描述受眾的知識背景（「懂 Python 但不熟機器學習」比「技術人員」更有效）
- 可描述受眾的目標或動機（他們想解決什麼問題）
- 可描述受眾的閱讀情境（快速瀏覽 vs. 深入學習）

---

## 提示詞範例

### 範例 1：向非技術主管說明技術決策

```
目標受眾是公司的業務副總，他具備商業決策經驗但沒有工程背景，
關注點在於成本、風險與商業影響，不需要技術細節。

請用他能理解的語言，解釋為何我們需要將系統從單體架構遷移至微服務架構，
並說明這個決策對業務的影響與預期效益。
```

---

### 範例 2：針對初學者的程式教學

```
目標受眾是剛開始學習程式設計的大學新生，從未接觸過任何程式語言，
但有基本的數學概念。

請用日常生活中的比喻，解釋什麼是「變數」與「函式」，
避免使用專業術語，並提供一個簡單的實際例子。
```

---

### 範例 3：產品說明頁面文案

```
目標受眾是 25-35 歲的上班族，對健康有興趣但生活忙碌，
習慣用手機快速瀏覽內容，注重實用性而非理論。

請為這款「智慧水瓶」撰寫產品說明，
強調它如何融入他們的日常生活並解決忘記喝水的問題。
```

## 常見錯誤

- **受眾描述過於籠統**：「一般大眾」無法有效引導模型調整深度
- **忽略受眾的目標**：只描述身份而未說明受眾想從內容中得到什麼
- **與 Persona 混淆**：Audience Persona 是「誰在讀」，不是「模型是誰」
