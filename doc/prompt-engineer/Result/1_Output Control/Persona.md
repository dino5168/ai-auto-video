# Persona（角色設定）

## 說明

Persona 範式是指在提示詞中明確指定模型應扮演的角色、身份或專業背景，藉此影響模型的語氣、知識框架、回答風格與決策視角。

透過角色設定，模型會從該角色的專業知識與思維方式出發，產生更具一致性與針對性的輸出。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 需要專業領域知識 | 希望模型以醫師、律師、工程師等專家視角回答問題 |
| 調整語氣與風格 | 希望模型以導師、朋友、顧問等不同關係角色溝通 |
| 模擬特定人物觀點 | 需要從歷史人物、虛構角色或特定職位者的角度分析 |
| 保持對話一致性 | 在多輪對話中維持固定的專業身份與語氣 |

---

## 使用提示

- 角色描述越具體，輸出越精準（避免只說「你是專家」）
- 可搭配限制條件，例如「不得超出該角色的知識範疇」
- 若需要多重角色，可明確指定主要角色與輔助角色

---

## 提示詞範例

### 範例 1：資深後端工程師

```
你是一位擁有 10 年以上經驗的資深後端工程師，專精於 Go 語言與分散式系統設計。
請從系統可靠性、效能與維護性的角度，審查以下程式碼並提出改善建議。

程式碼：
[貼上程式碼]
```

---

### 範例 2：法律顧問

```
你是一位台灣的企業法律顧問，熟悉公司法與勞動基準法。
請以法律風險的角度，分析以下合約條款可能存在的問題，並提出修改建議。

合約條款：
[貼上條款內容]
```

### 範例 3：UX 設計師

```
你是一位專注於行動裝置體驗的 UX 設計師，重視操作直覺性與使用者心智模型。
請針對以下應用程式的操作流程，指出使用者可能遭遇的摩擦點，並提出改善方向。

操作流程描述：
[貼上描述]
```

## 常見錯誤

- **角色過於模糊**：「你是一個助手」無法有效引導輸出風格
- **角色與任務不符**：設定了廚師角色卻要求分析法律文件，造成衝突
- **忽略角色限制**：未說明角色在哪些方面不應發表意見（例如角色應避免的立場）
