# Style Transfer（風格遷移）

## 說明

Style Transfer 範式是指在提示詞中要求模型模仿特定的文體風格、語氣或寫作特色，包括正式、口語、學術、幽默、特定作者風格等。

與 Persona 不同：Persona 改變的是「說話者的身份」，Style Transfer 改變的是「表達的形式與風格」。兩者可以組合使用。

---

## 使用時機

| 情境 | 說明 |
|------|------|
| 品牌語氣一致性 | 確保所有內容符合品牌的溝通風格手冊 |
| 內容改寫 | 將技術文件改寫為輕鬆易讀的部落格文章 |
| 特定平台適配 | 為 LinkedIn、Twitter、Email 等不同平台調整語氣 |
| 仿寫練習 | 模仿特定作者的寫作風格進行創作 |
| 文案 A/B 測試 | 同一內容用不同風格呈現，供比較測試 |

---

## 使用提示

- 可提供風格範本（「請模仿以下段落的風格：[範例]」）
- 可用形容詞描述風格（簡潔、溫暖、犀利、帶有幽默感）
- 可指定需要避免的風格元素（「避免使用專業術語」「不要過於正式」）
- 可同時指定多個風格維度（語氣 + 句型長度 + 用詞偏好）

---

## 提示詞範例

### 範例 1：改寫為輕鬆口語風格

```
請將以下技術說明改寫為輕鬆、口語化的部落格文章風格。
目標是讓對技術不熟悉的讀者也能輕鬆閱讀，
使用短句、適當的口語詞彙，可以加入一些親切的比喻。

原文：
[貼上技術說明]
```

---

### 範例 2：模仿學術論文風格

```
請以學術論文摘要的風格，重新撰寫以下研究發現。
要求：
- 使用被動語態
- 避免第一人稱
- 用詞精確，避免口語化表達
- 段落結構：背景 → 方法 → 結果 → 結論

研究發現：
[貼上內容]
```

---

### 範例 3：品牌語氣套用

```
我們品牌的語氣風格如下：
- 友善但專業，像一位可信賴的朋友
- 使用正向語言，避免否定句型
- 句子簡短有力，避免長句
- 偶爾使用問句引發讀者思考

請用上述風格，為我們的新功能「自動備份」撰寫一段 80 字以內的產品介紹。
```

---

### 範例 4：模仿特定作者風格

```
請模仿海明威的寫作風格（簡短有力的句子、直接的動詞、極少形容詞、隱藏情感於行動中），
為以下場景寫一段 150 字左右的描述：

場景：一個工程師在深夜獨自修復生產環境的嚴重錯誤。
```

---

## 常見錯誤

- **風格描述不具體**：「用好的風格寫」無法引導模型做出有意義的調整
- **風格與受眾不匹配**：使用學術風格寫給社群媒體受眾，效果適得其反
- **未提供風格參考**：對於模糊的風格需求，提供範文範例比描述更有效
